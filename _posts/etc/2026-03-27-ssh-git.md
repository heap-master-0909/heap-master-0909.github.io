## RSA vs ED25519

최근 Git/SSH에서 RSA 지원이 종료되었다. 이제 ED25519로 키를 생성하자.

```bash
ssh-keygen -t ed25519 -C "{email}"
```

생성하면 아래 두 파일이 만들어진다.

| 파일 | 설명 |
|------|------|
| `~/.ssh/id_ed25519` | 개인키 (절대 공유 금지) |
| `~/.ssh/id_ed25519.pub` | 공개키 (GitHub에 등록) |

---

## GitHub에 공개키 등록

```bash
# 공개키 내용 출력
cat ~/.ssh/id_ed25519.pub
```

출력된 내용을 복사해 **GitHub → Settings → SSH and GPG keys → New SSH key**에 붙여넣는다.

등록 후 연결 테스트:

```bash
ssh -T git@github.com
# Hi {username}! You've successfully authenticated. 가 나오면 성공
```

## 기본 키 설정 (Default Key)

`~/.ssh/config` 파일을 생성하고 아래를 추가한다.

```
Host *
  IdentityFile {private-key}
```

## 여러 GitHub 계정 사용 (Multi Account)

회사/개인 계정을 동시에 쓸 때 `~/.ssh/config`에서 Host 별칭을 나눈다.

```
# 개인 계정
Host github-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal

# 회사 계정
Host github-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work
```

이후 remote URL을 별칭으로 지정한다.

```bash
# 개인 repo clone
git clone git@github-personal:{username}/{repo}.git

# 회사 repo의 remote 변경
git remote set-url origin git@github-work:{org}/{repo}.git
```

## Git에서 SSH 사용

### 전역 설정 (Global)

```bash
git config --global core.sshCommand "ssh -i '{private-key}'"
```

### 특정 Repo에만 적용

`.git/config`에 아래를 추가한다.

```ini
[core]
  sshCommand = ssh -i '{private-key}'
```

## SSH Agent 활용

매번 passphrase 입력이 번거롭다면 ssh-agent에 키를 등록한다.

```bash
# agent 시작
eval "$(ssh-agent -s)"

# 키 등록
ssh-add ~/.ssh/id_ed25519

# 등록된 키 목록 확인
ssh-add -l
```

> Windows에서는 `Get-Service ssh-agent | Set-Service -StartupType Automatic` 후 `Start-Service ssh-agent`로 agent를 활성화한다.

## 사용 중인 키 확인

```bash
# 기본 확인
ssh -vT {git-url}

# 포트까지 확인
ssh -vT -p {port} {git-url}
```

## 방화벽으로 22번 포트가 막혔을 때

일부 네트워크에서 22번 포트가 차단된 경우 GitHub은 443 포트로도 SSH를 지원한다.

`~/.ssh/config`에 추가:

```
Host github.com
  HostName ssh.github.com
  Port 443
  User git
```
