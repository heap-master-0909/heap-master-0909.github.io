## Node.JS란?

Node.JS는 크롬 **V8 자바스크립트 엔진**으로 빌드된 **자바스크립트 런타임**이다.

### 런타임(Runtime)이란?

프로그래밍 언어가 실제로 동작하는 **실행 환경**을 의미한다.

| 언어 | 런타임 | 비고 |
|------|--------|------|
| Java | JRE (Java Runtime Environment) | JDK 안에 JRE가 포함됨 |
| JavaScript (브라우저) | 브라우저 엔진 (V8, SpiderMonkey 등) | 브라우저 내부에서만 실행 |
| JavaScript (서버) | **Node.JS** | 브라우저 밖에서 JS를 실행 |

> Java에서 코드를 실행하려면 JRE가 필요하듯, JavaScript를 브라우저 밖에서 실행하려면 **Node.JS**가 필요하다.

## 내부 구조

```
┌─────────────────────────────┐
│      Node.js Core Library   │  ← fs, http, path 등 내장 모듈
├─────────────────────────────┤
│      Node.js Bindings (C++) │  ← JS와 C/C++ 라이브러리를 연결
├──────────────┬──────────────┤
│     V8       │    libuv     │
│ (JS 엔진)    │ (비동기 I/O) │
└──────────────┴──────────────┘
```

| 구성 요소 | 역할 |
|-----------|------|
| **V8** | Google이 만든 고성능 JS 엔진. JS 코드를 기계어로 컴파일하여 실행 |
| **libuv** | 비동기 I/O를 처리하는 C 라이브러리. 이벤트 루프, 스레드 풀 제공 |
| **Core Library** | `fs`, `http`, `path`, `os` 등 서버 개발에 필요한 내장 모듈 |
| **Bindings** | JS 코드에서 C/C++ 라이브러리를 호출할 수 있게 연결하는 계층 |

## Node.JS 특징

### 1. 이벤트 기반 (Event-Driven)

이벤트가 발생했을 때 미리 등록해 둔 콜백 함수를 실행하는 방식이다.

```javascript
const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('greet', (name) => {
  console.log(`Hello, ${name}!`);
});

emitter.emit('greet', 'Node.JS');
```

### 2. 논블로킹 I/O (Non-Blocking I/O)

I/O 작업(파일 읽기, 네트워크 요청 등)이 완료될 때까지 기다리지 않고, 다음 코드를 먼저 실행한다. 작업이 끝나면 콜백으로 결과를 받는다.

```javascript
const fs = require('fs');

// 논블로킹 (비동기)
fs.readFile('file.txt', 'utf8', (err, data) => {
  console.log(data);  // 파일 읽기 완료 후 실행
});
console.log('파일 읽기 요청 완료');  // 이 줄이 먼저 실행됨
```

### 3. 싱글 스레드 + 이벤트 루프

Node.JS는 **싱글 스레드**로 동작하지만, **이벤트 루프**를 통해 비동기 작업을 효율적으로 처리한다.

```
   ┌───────────────────────┐
   │      Call Stack        │  ← JS 코드 실행
   └──────────┬────────────┘
              ▼
   ┌───────────────────────┐
   │     Event Loop         │  ← 완료된 비동기 작업의 콜백을 Call Stack에 전달
   └──────────┬────────────┘
              ▼
   ┌───────────────────────┐
   │  Task Queue / libuv   │  ← 파일 I/O, 네트워크 등 비동기 작업 대기
   └───────────────────────┘
```

> 싱글 스레드이므로 CPU 집약적인 작업(이미지 처리, 복잡한 연산)에는 적합하지 않다.

### 4. npm (Node Package Manager)

세계 최대 규모의 오픈소스 패키지 생태계를 제공한다.

```bash
npm init -y          # 프로젝트 초기화 (package.json 생성)
npm install express  # 패키지 설치
```

## Node.JS가 적합한 경우 vs 부적합한 경우

| 적합 | 부적합 |
|------|--------|
| 실시간 채팅, 스트리밍 | CPU 집약적 연산 (영상 인코딩 등) |
| REST API 서버 | 복잡한 비즈니스 로직이 많은 엔터프라이즈 앱 |
| 마이크로서비스 | 멀티 스레드가 필수인 작업 |
| IoT 데이터 수집 | |
