## REPL이란?

**REPL**은 **R**ead – **E**val – **P**rint – **L**oop의 약자로, 코드를 한 줄씩 입력하면 즉시 실행 결과를 보여주는 대화형 환경이다.

```
Read  → 사용자가 입력한 코드를 읽음
Eval  → 코드를 평가(실행)
Print → 결과를 화면에 출력
Loop  → 위 과정을 반복
```

### 실행 방법

터미널에서 `node`만 입력하면 REPL 모드로 진입한다.

```bash
$ node
Welcome to Node.js v20.x.x.
Type ".help" for more information.
>
```

```javascript
> 1 + 2
3
> 'Hello' + ' Node'
'Hello Node'
> const arr = [1, 2, 3]
undefined
> arr.map(x => x * 2)
[ 2, 4, 6 ]
```

> `undefined`가 출력되는 이유: 변수 선언(`const`, `let`)은 반환 값이 없으므로 `undefined`를 표시한다.

## CommonJS 모듈 시스템

Node.JS는 파일 하나가 곧 **하나의 모듈**이다. 모듈을 나누면 코드를 재사용하고, 유지보수하기 쉬워진다.

### 모듈이 필요한 이유

하나의 파일에 모든 코드를 작성하면 다음과 같은 문제가 생긴다.

- 파일이 커질수록 원하는 코드를 찾기 어려움
- 변수·함수 이름이 충돌할 위험 증가
- 코드 재사용이 불가능 (복사-붙여넣기)
- 여러 명이 동시에 작업하기 어려움

모듈 시스템은 이 문제를 해결한다.

```
┌── math.js ──────────────┐     ┌── app.js ────────────────┐
│ function add(a, b) {    │     │ const { add } = require  │
│   return a + b;         │ ──→ │   ('./math');             │
│ }                       │     │ console.log(add(2, 3));   │
│ module.exports = { add };│     └──────────────────────────┘
└─────────────────────────┘
```

### module.exports와 require

`module.exports`로 내보내고, `require()`로 가져온다.

```javascript
// greet.js — 모듈 정의
function greet(name) {
  return `안녕하세요, ${name}님!`;
}

module.exports = greet;
```

```javascript
// app.js — 모듈 사용
const greet = require('./greet');

console.log(greet('홍길동'));  // 안녕하세요, 홍길동님!
```

> `require`에 `./`를 붙이면 **같은 디렉터리의 파일**, 붙이지 않으면 **내장 모듈 또는 node_modules 패키지**를 찾는다.

### 다양한 내보내기 패턴

```javascript
// 1. 함수 하나를 통째로 내보내기
module.exports = function add(a, b) {
  return a + b;
};

// 2. 객체로 여러 함수 내보내기
module.exports = {
  add(a, b) { return a + b; },
  sub(a, b) { return a - b; },
};

// 3. exports 축약 — module.exports의 프로퍼티에 하나씩 추가
exports.add = (a, b) => a + b;
exports.sub = (a, b) => a - b;
```

#### exports vs module.exports

`exports`는 `module.exports`를 가리키는 **참조(reference)**다. 따라서 `exports` 자체를 재할당하면 연결이 끊어진다.

```javascript
// ❌ 동작하지 않음 — exports 참조가 끊어짐
exports = { add: (a, b) => a + b };

// ✅ 올바른 방법
module.exports = { add: (a, b) => a + b };
```

```
┌─────────────────────────────────────────┐
│ module.exports = {}   ← 최종 내보내기   │
│       ▲                                 │
│       │ (참조)                           │
│    exports                              │
│                                         │
│ exports.add = fn  → module.exports에도   │
│                     add가 추가됨 ✅      │
│                                         │
│ exports = { ... } → 참조가 끊어짐 ❌     │
└─────────────────────────────────────────┘
```

### 구조 분해로 가져오기

```javascript
// math.js
module.exports = {
  add: (a, b) => a + b,
  sub: (a, b) => a - b,
  mul: (a, b) => a * b,
};
```

```javascript
// app.js — 필요한 함수만 골라서 가져올 수 있다
const { add, mul } = require('./math');

console.log(add(2, 3));  // 5
console.log(mul(4, 5));  // 20
```

### require의 모듈 탐색 순서

`require('모듈명')`을 호출하면 Node.JS는 아래 순서로 모듈을 찾는다.

```
1. 코어 모듈 (fs, path, http 등 Node.JS 내장)
2. ./  또는 ../ 로 시작하면 파일/디렉터리 경로로 해석
3. node_modules/ 디렉터리 (현재 → 상위 → … → 루트까지 탐색)
```

```javascript
const fs = require('fs');           // 1. 코어 모듈
const myLib = require('./lib/my');  // 2. 상대 경로
const express = require('express'); // 3. node_modules
```

파일 확장자를 생략하면 `.js` → `.json` → `.node` 순서로 시도한다.

### 모듈 캐싱

`require()`로 불러온 모듈은 **캐싱**된다. 같은 모듈을 여러 번 `require`해도 **최초 1회만 실행**되고, 이후에는 캐싱된 결과를 반환한다.

```javascript
// counter.js
let count = 0;
module.exports = {
  increment() { return ++count; },
  getCount() { return count; },
};
```

```javascript
// app.js
const counter1 = require('./counter');
const counter2 = require('./counter');  // 같은 인스턴스

counter1.increment();
counter1.increment();
console.log(counter2.getCount());  // 2 — counter1과 같은 객체
```

> 캐싱 덕분에 싱글턴 패턴처럼 동작한다. 캐시를 확인하려면 `require.cache`를 출력해 보자.

## ES 모듈 (ESM)

ES2015(ES6)에서 표준화된 모듈 시스템이다. 브라우저와 Node.JS 모두에서 사용 가능하며, Node.JS v12+부터 안정적으로 지원된다.

### 사용 조건

ESM을 사용하려면 다음 중 하나를 충족해야 한다.

| 방법 | 설명 |
|------|------|
| 파일 확장자 `.mjs` | 파일 단위로 ESM 적용 |
| `package.json`에 `"type": "module"` | 프로젝트 전체에 ESM 적용 |

### import / export

```javascript
// math.mjs (또는 type: "module"인 프로젝트의 math.js)

// named export
export function add(a, b) {
  return a + b;
}

export function sub(a, b) {
  return a - b;
}

// default export
export default function mul(a, b) {
  return a * b;
}
```

```javascript
// app.mjs
import mul, { add, sub } from './math.mjs';

console.log(add(2, 3));  // 5
console.log(sub(5, 2));  // 3
console.log(mul(4, 5));  // 20
```

### CommonJS vs ES 모듈 비교

| 비교 항목 | CommonJS (`require`) | ES 모듈 (`import`) |
|-----------|---------------------|---------------------|
| 로딩 시점 | **런타임** (동적) | **파싱 타임** (정적) |
| 문법 | `require()` / `module.exports` | `import` / `export` |
| 트리 쉐이킹 | 어려움 | 가능 (정적 분석) |
| 비동기 로딩 | 지원하지 않음 | `import()` 동적 임포트 지원 |
| 최상위 await | 불가 | 가능 (Node.JS 14.8+) |
| 파일 확장자 | `.js` (기본) | `.mjs` 또는 `"type": "module"` |
| Node.JS 지원 | 처음부터 지원 | v12+ (안정적) |

```javascript
// CommonJS — 런타임에 조건부 로딩 가능
if (condition) {
  const lib = require('./lib');
}

// ESM — 정적이므로 최상위에서만 import
import { lib } from './lib.mjs';

// ESM — 동적 import()로 조건부 로딩 가능
if (condition) {
  const lib = await import('./lib.mjs');
}
```

> 새 프로젝트라면 **ESM**을 권장한다. 다만 기존 Node.JS 생태계의 많은 패키지가 CommonJS이므로 두 방식 모두 이해해 두어야 한다.

## 주요 내장 모듈

Node.JS는 별도 설치 없이 사용할 수 있는 **내장(코어) 모듈**을 제공한다.

### 자주 사용하는 내장 모듈

| 모듈 | 용도 | 예시 |
|------|------|------|
| `fs` | 파일 읽기/쓰기 | `fs.readFile()`, `fs.writeFile()` |
| `path` | 경로 처리 | `path.join()`, `path.resolve()` |
| `http` | HTTP 서버/클라이언트 | `http.createServer()` |
| `os` | 운영체제 정보 | `os.cpus()`, `os.platform()` |
| `url` | URL 파싱 | `new URL()` |
| `events` | 이벤트 처리 | `EventEmitter` |
| `crypto` | 암호화 | `crypto.createHash()` |
| `util` | 유틸리티 | `util.promisify()` |
